# 编排模式

多 Agent 系统的协调模式。

## 四种核心模式

### Sequential Pipeline（顺序管道）
一个 Agent 的输出作为下一个 Agent 的输入。适用于线性工作流。

### Parallel Fan-out（并行分发）
多个 Agent 同时处理独立子任务，结果合并。适用于需要并行探索的场景。

### Hierarchical Supervisor（层级Supervisor）
Supervisor Agent 负责任务分解和委托给 Worker Agents。

### Team Pipeline（团队管道）
多阶段管道，每个阶段可能启动多个 Agent。参见 [[01-核心知识/Agent编排/AI_Agent_Orchestration]] 中的 OMC team 模式。
