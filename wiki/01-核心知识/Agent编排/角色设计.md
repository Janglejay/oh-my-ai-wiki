# 角色设计

多 Agent 系统中的专业化角色定义。

## 常见 Agent 角色

| 角色 | 职责 |
|------|------|
| **Explorer** | 代码库发现、文件/符号映射 |
| **Planner** | 任务排序、执行计划制定 |
| **Architect** | 系统设计、接口定义 |
| **Executor** | 代码实现、重构 |
| **Verifier** | 完成验证、测试确认 |
| **Reviewer** | 代码审查、安全审查 |
| **Critic** | 差距分析、计划挑战 |

## 角色设计原则

1. **单一职责** — 每个角色专注于特定类型的任务
2. **清晰边界** — 角色之间的交互接口明确
3. **能力匹配** — 角色能力与任务复杂度匹配
